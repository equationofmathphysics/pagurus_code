# 🦀 PaGURUS 核心概念

## 三句话讲清楚

**是什么**: 一个基于"模板+执行+验证"的仓库级代码 Agent  
**怎么做**: 找壳 → 钻壳 → 改壳 (检索模板 → 跑通环境 → 增量开发)  
**为什么**: 拒绝 Zero-shot 盲目生成，站在开源库肩膀上写代码

---

## 核心隐喻

```
🦀 寄居蟹 = 代码 Agent
🐚 壳     = 优秀开源项目
🔨 钻壳   = 跑通环境 + 自动修复
✏️ 改壳   = 理解结构 + 增量开发
```

---

## 工作流程

```
用户需求 
   ↓
寻壳: 检索最匹配的开源模板
   ↓
钻壳: Clone + 调试直到能运行
   ↓
改壳: 理解项目结构 + 精准修改
   ↓
验证: 编译运行 + 报错自修复
   ↓
交付: 完整可运行的项目
```

---

## 与传统 Agent 的区别

| | 传统 LLM Agent | PaGURUS |
|-|---------------|---------|
| 起点 | 从零生成 | 基于成熟模板 |
| 验证 | 不验证 | 沙盒真实执行 |
| 修复 | 无闭环 | 报错自动修复 |
| 质量 | 不可控 | 有下限保障 |

---

## 研发路径

- **PoC (1周)**: 固定模板 + 手动调试
- **MVP (1月)**: 自动检索 + 沙盒 + 闭环
- **Research (3月)**: 环境自修复 + 仓库级理解
